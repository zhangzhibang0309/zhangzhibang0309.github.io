hexo-next主题博客搭建中踩过的坑

发表于 2020-01-27 更新于 2020-02-02 分类于 教程 阅读次数：
本文字数： 6.3k 阅读时长 ≈ 6 分钟

前言

图是灵魂

不知道说什么也要说

　　从开始搭建博客到对主题进行各种配置，遇到了各种各样的问题，由于版本不一样导致网上的教程并不适用，期间踩过各种坑，这里我就把我搭建博客中用到的小技巧以及各种问你题我是怎么解决的总结一下。

hexo建站–本文部署到Github

下载内容

我们要安装的内容还是不少的
1.node.js,在他的英文官网下载是很慢的，几乎下不下来，这个是它的中文网，下载速度很快。下载对应官网LTS版。
2.Git,点击此处下载Git。
3.Hexo,在官网下载Hexo。
4.HBuilder,这是我比较喜欢用的代码编辑器，大家可以用这个，为以后主题配置系列操作准备。
这里具体建站以及部署到本地我就不细说了，我是跟着小破站上的视频照着做的，期间没遇到什么问题。

部署到github

大家可根据小破站视频操作到部署到本地，这时候我们的hexo已经基本搭建起来，但这只是完成了一小部分，后面的主题配置，别人不知道，我可是要抓狂了。那么，首先我们要申请github账号，建立自己的仓库，还要配置githubssh，一顿操作后，部署的准备工作基本做完了。接下来就是踩坑了。
1.网站404
在_config.yml中
deploy:
type: git
repo: https://github.com/账号名称/仓库名称.github.io.git
branch: master

以及上面的url: https://仓库名.github.io
配置完这些，我满怀期待的打开url后面的网址，结果404将我绝杀。然后开始查各种资料，后来终于找到了问题，在github上存放博客文件的仓库名称必须与账号同名。这样就可以解决了。

2.推送到github失败
这张图相信像我这样的新手小白应该不会陌生了，
推送的流程是
hexo clean
hexo d -g
然后就出现了这个可恨的Spawn failed，当然最好推送的时候不要使用cmd，用git bash容易一些，不过，还是会出错，最近不知道是家里网的原因还是怎么，github一直进不去，查看密钥也提示time out，与提交现实的错误相同，这个错误折磨了我好长一段时间，也从网上查了各种各样的原因，像我这样的萌新肯定也听说过网上的各种解决方案，什么修改端口，还有更改代理什么的，以及更改deploy的配置，还有一个说是密钥出问题了，这些除了设置代理我都试过了，然并卵，最后我在看与github为什么进不去的一个帖子时候，无意间看到了有人用4G可以进去，我就没有报多大的希望，就用手机开了热点，试了一下，神了！再也没出现过问题。
Ps:如果是会文明上网的大佬应该不会存在此问题。

Next主题配置相关问题

温馨提示

这个Next主题的机制大概是这样的，主题本身就提供了很多功能，在主题配置文件中很多功能后面把false改成true即可开启此功能。

更换Next主题

Next是非常出名也是用的最多的一款主题，hexo官网有很多主题，我选择next的原因主要是next教程比较多(原谅我菜)。大家可以去GitHub直接搜索next，点击第一个项目，去下载，然后把next文件放到本地博客根目录的themes里面，然后再配置文件(_config.yml)中theme: 后面更改为next，打开cmd，进入博客根目录，hexo s -p，部署到本地，就可以看到，你的博客已经完全变样子了，类似于换了一个皮肤。
然后，在主题配置文件，就是在next文件夹下面的(_config.yml)中有个scheme配置，在next之下还有四种样式可以选择。我个人用的是Pisces。
好，到这里，大概你的博客除了丑没什么其他问题了。

博客基本信息配置

在配置文件中
title: 博客名称
subtitle: ‘’
description: ‘’
keywords:
author: 作者
language: zh-CN
timezone: ‘’
以上信息，主要的我都已经标注了，language默认配置是英文，改成zh-CN就可以将博客语言改成中文了。
接下来就是要安排头像了，作为个人博客，怎么能没有头像呢。
配置方法：在next下的images中放入你想要作为头像的图片，进入主题配置文件，找到如下配置
avatar:
url: /images/头像.jpg
按照此模板配置好即可，因为我发现hexo的图片链接容易炸，所以就不要自己建文件夹放图片了，按照我说的应该是没问题的，我都已经测试过。这下，头像也就布置好了。

菜单目录配置

现在你的博客中只有菜单目录只有两个选项，首页，和归档，添加其他的选项是必须的，在主题配置文件中，找到
menu:
home: / || home 主页
about: /about/ || user 关于
tags: /tags/ || tags 标签
categories: /categories/ || th 分类
archives: /archives/ || archive 归档
schedule: /schedule/ || calendar 日程表
#sitemap: /sitemap.xml || sitemap 忘了是啥了
#commonweal: /404/ || heartbeat 404爱心
可根据自己的喜好，将注释去掉，就可以开启了，开启之后，在博客根目录下的source建立文件夹，每个文件夹名字对应上面的about，tags，只建立分类，关于，标签就可以了，然后再在文件夹里新建index.md，在里面加入
---
title: 如下分类
date: 2019-06-25 19:16:17
type: “categories”
---
type与文件夹名称对应，时间随便，title就是你点进分类后现实的一句话，只有添加这个之后，点进去分类选项之后，才会显示各个选项，否则你虽然给文章添加了分类，无法通过点击菜单栏中的分类进行查看。然后，在你写博文的时候，要把下面这些添加在文首
---
title: 文章名称
date: 2020-01-27 17:04:59
categories: 归为什么分类
tags: [标签1,标签2,标签3,标签4]
---
这样就完成了对于菜单栏的一系列配置。

添加社交链接

在主题配置文件中找到social:，后面有一系列社交网站，想添加哪个，去掉注释即可，并把后面的网站改成相应的网站就行了，到这里都是比较愉悦的，大坑还在后面。

添加访问人数，访问次数

在主题配置文件中
busuanzi_count:
enable: true
total_visitors: true
total_visitors_icon: user
total_views: true
total_views_icon: eye
post_views: true
post_views_icon: eye
把enable后面改成true即可在博客底部看到访问人数|访问次数。

添加live2d看板娘

首先根目录下安装插件

接下来安装模型

模型可以从百度上查，以后我可以写个博客把模型展示一下。
安装完成之后，在根目录下，有个node_modules,找到live2d-widget-model-模型名称，然后把这个文件夹复制到根目录下的live2d_models中，再去主题配置文件中找到
live2d:
enable: true
scriptFrom: local
pluginRootPath: live2dw/
pluginJsPath: lib/
pluginModelPath: assets/
tagMode: false
debug: false
model:
use: live2d-widget-model-黑猫
display:
position: right
hOffset: -40 #水平位置
vOffset: -30 #垂直位置
width: 200
height: 350
mobile:
show: true
最上面设置为true，use后面是想改成的模型的名称，也就是刚才复制到live2d_models中的文件加名称。
我想说几个自定义小技巧，hoffset，voffset可是调整模型位置，width和height可以调整大小。

修改背景图片

这个是我一开始就想做的，背景一片白确实不怎么好看，可是又是因为next新版本的原因，没找到一个修改背景图片的方法，那只能自己动手丰衣足食了。
开检查查看代码的时候，我发现最后面的背景的css为body，位置不用说也知道，在next的css文件夹中，我找了半天，在next，source的css中，scaffolding中有个base.styl中，找到body
添加以下代码
background:url();
background-repeat:no-repeat;
background-attachment:fixed;
background-position:50% 50%;
background-size:cover;
把之前的background给删掉，url后面添加图片地址，我试过集中本地链接，不知道为什么，都炸了，我看到网上有什么自动换背景图片，我试了试真行，然后就从网上找了几张图试了试确实可以，然后就想到了用图床，点击图床把本地图片提交，就可以得到图片的网络连接。
然后我在说一下下面的配置，主要是最后这个，如果没有这个的话，手机端是直接没法看的，加上这个手机端也不是很好搞看，我还没研究出来怎样适配手机端，就先这样吧。

对于图片问题的解决

我在上传图片时遇到的问题

![](img/1.jpg)，img是我在与文章相同的子目录中，直接运行markdown的话是可以显示的，然后在本地查看博客或者是推送到github上都是无法显示的。

解决方法

通过询问度娘最也解决了。

1.首先要把_config.yml中的post_asset_folder: false，要把false改成true，修改之后才可以上传图片。

2.要安装一个插件，我操作的时候过程有点乱，也不确定装没装上，不过看网上说需要安装这个插件，npm install hexo-asset-image –save，要在博客的根目录下运行这条指令

3.这是我烦的主要问题，我的链接图片的格式不对![](img/1.jpg)，不能将图片放在与文章相同子目录，正确的方式是在source中新建一个文件夹将其命名为images，把图片放在里面，连接的时候格式是![](/images/1.jpg)，第一个“/”的意思是根目录，也就是代表了source。其他格式的链接都不会成功，我尝试的只有这种情况在在博客中图片是可以显示的。具体原因不明。

4.完成这些步骤之后，运行blog就可以看到图片了。
